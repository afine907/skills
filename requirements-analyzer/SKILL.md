# 需求梳理分析技能

## 技能概述

本技能帮助用户将模糊的产品需求转化为结构化的需求文档，输出包含模块分类、优先级和用户故事的专业需求文档。

## 输入要求

用户应提供：
1. **核心需求点**：产品要解决的核心问题、目标用户群体、核心价值主张
2. **用户旅程**：用户使用产品的完整流程步骤
3. **可选补充**：竞品信息、约束条件、特殊需求

## 输出文档结构

```
# [产品名称] 需求文档

## 一、项目概述
- 背景与目标
- 目标用户
- 核心价值

## 二、需求模块概览
| 模块名称 | 功能数量 | 优先级 | 状态 |
|----------|----------|--------|------|
| 用户模块 | 5 | P0 | 待开发 |
| 核心业务 | 8 | P0 | 待开发 |
| ... | ... | ... | ... |

## 三、用户故事地图
[按用户旅程阶段组织的故事列表]

## 四、需求详细说明
### 模块一：[模块名称]
#### P0（核心功能）
- 需求编号: REQ-XXX
- 需求描述: [简要描述]
- 用户故事: As a [用户角色], I want to [功能], so that [价值]
- 功能点:
  - [功能点1]
  - [功能点2]
- 验收标准:
  - [标准1]
  - [标准2]

#### P1（重要功能）
...

#### P2（增强功能）
...

## 五、页面/流程规划
[关键页面和交互流程]

## 六、风险与依赖
[已知风险和外部依赖]
```

## 输出规范

### 优先级定义

| 优先级 | 定义 | 说明 |
|--------|------|------|
| **P0** | 核心功能 | 产品必须具备的功能，无此功能无法使用 |
| **P1** | 重要功能 | 提升用户体验，核心流程的必要补充 |
| **P2** | 增强功能 | 锦上添花，MVP后可实现 |

### 用户故事模板

```
作为 <用户角色>，
我希望 <功能/行为>，
以便 <价值/收益>。

验收标准:
1. [场景1：成功路径]
2. [场景2：边界条件]
3. [场景3：异常处理]
```

### 模块分类原则

按业务领域或功能类型划分：
- **用户模块**：登录、注册、个人中心等
- **核心业务**：产品主要价值相关的功能
- **内容模块**：信息展示、搜索、推荐等
- **交易模块**：下单、支付、售后等
- **运营模块**：后台管理、数据分析等

## 工作流程

1. **需求解析**：识别核心需求点和用户旅程阶段
2. **模块规划**：按业务领域规划需求模块
3. **功能拆解**：将需求拆解为具体功能点
4. **优先级评估**：根据核心程度排序
5. **用户故事编写**：转换为标准用户故事格式
6. **输出文档**：生成结构化需求文档

## 质量检查清单

输出前检查：
- [ ] 覆盖所有用户旅程阶段
- [ ] P0功能完整且必要
- [ ] 用户故事符合INVEST原则
- [ ] 功能点可测试、可验收
- [ ] 无遗漏的异常流程

## 常见问题处理

| 场景 | 处理方式 |
|------|----------|
| 需求不清晰 | 列出假设，向用户确认 |
| 功能点过多 | 建议MVP范围，建议拆分 |
| 优先级冲突 | 提供排序理由，建议调整 |
| 缺少技术约束 | 标注为待确认项 |
