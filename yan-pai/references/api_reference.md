# Code Review Checklist

详细的代码审查清单，用于指导审查Agent进行全面检查。

## 1. 代码质量 (Code Quality)

### 1.1 可读性
- [ ] 变量和函数命名是否清晰、有意义
- [ ] 代码结构是否清晰，逻辑是否易于理解
- [ ] 是否有必要的注释解释复杂逻辑
- [ ] 代码格式是否一致（缩进、空格、换行）

### 1.2 函数设计
- [ ] 函数是否单一职责
- [ ] 函数长度是否合理（建议<50行）
- [ ] 参数数量是否合理（建议<5个）
- [ ] 是否有重复代码可以抽取

### 1.3 代码组织
- [ ] 文件结构是否合理
- [ ] 模块划分是否清晰
- [ ] 依赖关系是否合理

## 2. 潜在Bug (Potential Bugs)

### 2.1 边界条件
- [ ] 数组/列表是否检查越界
- [ ] 循环边界是否正确
- [ ] 条件判断是否覆盖所有情况

### 2.2 空值处理
- [ ] 是否检查null/undefined
- [ ] 可选属性是否正确处理
- [ ] 默认值是否合理

### 2.3 异常处理
- [ ] 是否捕获可能的异常
- [ ] 异常处理是否合理
- [ ] 错误信息是否有意义

### 2.4 并发安全
- [ ] 共享资源是否正确同步
- [ ] 是否存在竞态条件
- [ ] 死锁风险评估

## 3. 性能问题 (Performance)

### 3.1 算法效率
- [ ] 时间复杂度是否可接受
- [ ] 空间复杂度是否可接受
- [ ] 是否有不必要的循环嵌套

### 3.2 资源管理
- [ ] 是否有内存泄漏风险
- [ ] 连接/资源是否正确释放
- [ ] 缓存策略是否合理

### 3.3 数据处理
- [ ] 是否有不必要的数据复制
- [ ] 大数据处理是否优化
- [ ] 懒加载是否合理使用

## 4. 安全漏洞 (Security)

### 4.1 输入验证
- [ ] 用户输入是否验证
- [ ] 参数类型是否检查
- [ ] 边界值是否验证

### 4.2 注入防护
- [ ] SQL注入防护
- [ ] XSS防护
- [ ] 命令注入防护

### 4.3 敏感数据
- [ ] 敏感数据是否加密
- [ ] 日志是否泄露敏感信息
- [ ] 权限检查是否完整

## 5. 最佳实践 (Best Practices)

### 5.1 设计模式
- [ ] 是否合理使用设计模式
- [ ] 代码扩展性
- [ ] 代码可测试性

### 5.2 错误处理
- [ ] 错误是否优雅降级
- [ ] 是否有重试机制
- [ ] 错误日志是否完整

### 5.3 测试覆盖
- [ ] 单元测试覆盖率
- [ ] 边界条件测试
- [ ] 异常路径测试

## 严重性等级定义

| 等级 | 定义 | 示例 |
|------|------|------|
| Critical | 必须立即修复，会导致系统崩溃或严重安全漏洞 | SQL注入、空指针崩溃 |
| High | 应该尽快修复，影响核心功能或性能 | 内存泄漏、并发问题 |
| Medium | 建议修复，影响代码质量或可维护性 | 代码重复、命名不规范 |
| Low | 可选修复，轻微改进建议 | 注释优化、格式调整 |

## 评分标准

| 分数范围 | 状态 | 说明 |
|----------|------|------|
| 90-100 | 赌侠 | 代码质量优秀，无重大问题 |
| 70-89 | 接近赌侠 | 代码质量良好，有轻微问题 |
| 50-69 | 小瘪三 | 代码质量一般，需要改进 |
| 0-49 | 严重小瘪三 | 代码质量差，需要重构 |
