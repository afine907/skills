description: Analyze code core functionality, directory structure, and design quality. For explaining how code works, teaching codebases, or evaluating code design.
---

When analyzing code, always include:

1. **Core functionality**: What does this project do? Where is the entry point?
2. **Directory structure**: File organization, purpose of each directory
3. **Design evaluation**: What's good + what needs improvement
4. **Code explanation**: Use analogies, ASCII diagrams, step-by-step walkthrough
